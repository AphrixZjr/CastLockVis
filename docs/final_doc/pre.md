# CastLock-Vis · 设计陈述（设计动机 / 核心创新点 / 技术难点）

> 选题：演员的「类型锁定」与「转型窗口期」。
> 一套静态单页可视分析系统，沿 **宏观群落 → 中观序列 → 微观个体** 三级流水线，
> 用 **四个联动视图** 把「纯统计看不见的行业规律」可视化出来。

---

## 一、设计动机（Design Motivation）

> 回答一个问题：为什么值得专门做这样一套系统。

- **「类型锁定」是真实却没被看见的行业困局**
  演员被锁进舒适圈、错过转型窗口是普遍且代价高昂的现象，但它一直缺少一个能把「锁定」直观摊开来看的观察工具。

- **聚合统计只给结论，给不出「过程」**
  平均熵、转移概率这类数字能说「锁定确实存在」，却答不出它何时发生、谁能挣脱、又是怎么挣脱的——而这恰恰是问题的要害，必须靠可视分析逐层下钻才能逼近。

- **转型成败的「外部成因」需要被审视**
  对演员、经纪与投资方真正有决策价值的，是「什么外部条件促成了转型」；要回答它，就得把上千条个体轨迹与其控制变量并置对比，这是单纯建模做不到的。

---

## 二、核心创新点（Core Innovations）

> 强调相对现有做法的新意 / 新应用，而非给定约束下的常规工程选择。

- **把「事件对齐」迁移到类型生涯**
  借鉴生存分析 / 事件序列里「对齐到关键事件」的思想，自动捕捉每位演员「首部彻底偏离舒适圈」的作品并锚为 T=0——首次让转型时机各异的演员可被横向并排比较，这是该思想在演员类型生涯上的新应用。

- **用「类型偏离度」重定义转型坐标轴**
  现有做法默认用香农熵衡量「多元度」，但熵会把「成功演化」与「被弹回固化」压成同一条曲线；本系统改用 `dist = 1 − cos(早期向量, 滚动类型向量)` 这一自参照的方向性偏离度，才在度量层真正把两种结局分开。

- **软马尔可夫转移门保留「耦合片」信息**
  常规转移矩阵对每部片做硬主导分类，会丢掉跨类型片的混合性；本系统把每部片转成 IDF 归一化类型向量、相邻外积累加再归一，以软转移更真实地刻画「重力场」对角线如何随生涯坍缩。

## 三、技术难点（Technical Challenges）

> 重点说明「待解决的问题」，解决方法从简。

- **难点：无真值地判定「转型起点」与「成败」**
  既没有标注好的 T=0，也没有「成功 / 被弹回」的标签，且轨迹噪声大、极易把「冲高后又回升」误判为失败。
  *解法：* t0 取熵 onset 变点，`outcome` 用「偏离度回撤 + 末段斜率」双条件判定，并用聚类的类型语义交叉验证。

- **难点：数据分布让「自然的同侪定义」退化失效**
  按「同序号 / 同 tau」选同侪本是直觉做法，但 `t0Index` 73% 集中于 6、轨迹又都挤在 τ∈[−3,5] 窗口内，结果不是匹配到几乎全部就是几乎为零，联动失去意义。
  *解法：* 改以类型锁定群落 `clusterId` 界定同侪，兼顾选择性与语义。

- **难点：上千条轨迹随滑块实时重分层会卡顿**
  控制变量过滤要让 1000+ 条线即时重新分组、重新着色，逐条 `<path>` 渲染无法保证滑块拖动的即时响应。
  *解法：* 几何与分类分层 memo，并把同样式桶的上千条轨迹合并成单条 `<path>`（DOM 由约 1100 降到约 7）。
