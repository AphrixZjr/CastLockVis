# S1 约定暂存（待验收后同步主文档）

更新时间：2026-06-01

只保留会影响 S2+ 开发和维护的约定。

## 1) Markov 索引键规范

- `markovByClusterStage` 使用字符串键：`${cohortId}:${stage}`。
- 后续所有按阶段读取 Markov 矩阵的逻辑必须沿用此键规则。
- 当前读取路径使用 `${clusterId}:${stage}` 查询该索引。
- 上述查询成立依赖数据侧前提：`cohortId === clusterId`；若后续数据定义分离，必须新增显式映射层，不可继续隐式复用。

## 2) filmsByActor 的排序责任

- `buildIndexes()` 会把 `filmsByActor` 统一按 `seqIndex` 升序排序。
- 下游视图默认可直接消费有序电影序列，不重复排序。

## 3) cohort -> cluster 映射策略

- `getDominantClusterId()` 使用众数策略（出现次数最多的 `clusterId`）。
- A->D 联动当前按该策略映射到群落级矩阵。
- 众数并列时，固定选择数值更小的 `clusterId` 作为 tie-break，保证结果可复现。

## 4) alignmentFilters 默认语义

- `alignmentFilters` 默认值是 `[-Infinity, +Infinity]`。
- 该默认值语义是“不过滤协变量区间”。
- 当前过滤实现会固定排除 `outcome='none'` 轨迹，因此默认态并非“返回全部 alignment”。
- 后续任何过滤 UI 和聚合口径都必须显式声明是否包含 `outcome='none'`，避免默认行为歧义。

## 5) alignment 契约语义（判别联合）

- `alignment` 前端类型采用判别联合：
  - `outcome='none'`：`points` 为空，`covariatesAtT0` 为空语义值；
  - `outcome='success'|'snapback'`：`covariatesAtT0` 必须是数值字段。
- 使用 alignment 数据时应先按 `outcome` 分支，再读取协变量。

## 6) 数据加载分层

- 启动加载逻辑固定在 `data/DataProvider.tsx`。
- 业务视图层统一通过 `useDataRuntime()` 消费 `loading/error/ready` 三态，不直接请求契约文件。

## 7) 开发态请求去重

- `loadDataBundle()` 使用模块级 Promise 缓存。
- 目的：避免 React 18 StrictMode 开发态双挂载导致重复请求。
- 若加载失败会清空缓存，允许重试。

## 8) Toggle 控件复用约定

- 分段切换控件统一复用 `components/controls/Toggle`。
- 后续新增 Toggle 场景优先扩展该控件，避免重复实现。

## 9) ViewPanel 状态渲染规则

- 当 `status` 是 `loading/empty/error` 时，`ViewPanel` 渲染状态占位，不渲染 `children`。
- `children` 仅在 `status='ready'` 时渲染。
- 后续如需“内容+遮罩”并存，需先扩展组件契约，再接入视图逻辑。

## 10) 布局尺寸 token 扩充约定

- 已补充并使用：
  - `--panel-state-min-height`
  - `--panel-toolbar-min-height`
  - `--panel-legend-min-height`
  - `--panel-mobile-row-min-height`
- 新增面板尺寸优先走 token，不新增散落硬编码尺寸。

## 11) 已知技术债（暂不修复）

- `alignment` 判别联合中，`outcome='none'` 的 `covariatesAtT0` 在类型层当前仍为“可空数值字段”而非“仅 null 字段”。
- 当前数据与解析器会把该分支约束到空语义，不影响现阶段运行；后续若出现手写测试数据或运行态拼装对象，可能绕过该约束。
- 建议在后续类型清理阶段收紧该分支类型（仅允许 null），以降低调试成本。
